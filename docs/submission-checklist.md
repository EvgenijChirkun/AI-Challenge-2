# Submission Checklist

Use this checklist before submitting each task. Copy it into the relevant task's `README.md` or `docs/` folder and check off each item.

## Task Submission Checklist

### Code & Files
- [ ] Solution files added to `src/`
- [ ] All required files are present and correctly named

### Documentation
- [ ] Task `README.md` updated with title and description
- [ ] Chosen tech stack documented
- [ ] Architecture / approach described
- [ ] Run instructions written and tested locally
- [ ] Input / output format described
- [ ] Files and folders section updated

### Dependencies
- [ ] All dependencies documented (e.g., `requirements.txt`, `package.json`, `go.mod`, `Cargo.toml`, etc.)
- [ ] No global or undocumented dependencies required

### Data & Artifacts
- [ ] Data files added to `data/` or referenced with a URL / download instructions
- [ ] Required artifacts added to `artifacts/` (models, reports, outputs)

### Deployment / Demo
- [ ] Deployment config added to `deployment/` (if applicable)
- [ ] Demo or deployment link added to `README.md` (if applicable)

### Security & Hygiene
- [ ] No secrets, API keys, or passwords committed
- [ ] No large unnecessary binary files committed
- [ ] No private or sensitive data included

### Final Review
- [ ] Submission notes completed in task `README.md`
- [ ] Results documented
- [ ] All changes committed with meaningful commit messages
